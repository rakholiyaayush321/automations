# GitHub Copilot Instructions for Job Application Automation

## Project Context

This is an autonomous job application automation system for Ayush Rakholiya that:
- Scrapes jobs from 5 platforms (LinkedIn, Indeed, Internshala, Wellfound, Naukri)
- Scores jobs against AI/ML/Python skills
- Applies via email automatically to qualified candidates
- Tracks applications and prevents duplicates
- Allows reapplication after 30 days

## Code Style & Conventions

### Python Standards
- Python 3.9+
- Use type hints for all functions: `def load_jobs() -> list[dict]:`
- Import organization: stdlib → third-party → local
- Maximum line length: 88 characters
- Use f-strings for string formatting: `f"Company: {name}"`

### Error Handling
- Use specific exceptions, not bare `except:`
- Log errors with context: `logger.error(f"Failed to scrape {url}: {e}")`
- Always provide user-friendly error messages

### File Organization
- `config.py` - All configuration constants
- `jobs/` - Job scraper modules (one per platform)
- `matcher.py` - Skill matching algorithm
- `dedupe.py` - Duplicate detection
- `emailer_enhanced.py` - Email sending with validation
- `logger.py` (CSV logging) - Application tracking
- `main.py` - Orchestration and entry point

## Critical Implementation Details

### Matcher Algorithm
- Use word boundaries for skill matching: `r'\b' + skill + r'\b'`
- Don't use simple substring matching (causes false positives)
- Normalize text before matching: lowercase, remove extra whitespace
- Example: "python" should NOT match "pythonic" or "cpython"

### Deduplication
- Mark companies as applied BEFORE sending email (prevents race conditions)
- Use SHA256 hash of (company_name + job_title)
- Store in JSON for persistence

### Email Validation
- Check format with regex
- Detect system emails (noreply@, admin@, etc.)
- Validate domain exists
- Skip emails that previously failed

### Reapplication Logic (New Feature)
- Track application date in CSV
- Allow reapplication if 30+ days have passed since last attempt
- Company/title combo is reapplicable after 30 days
- Use `datetime` for date calculations


## Features to Respect

- **Daily target limit**: `DAILY_TARGET = 20` - Don't exceed 20 applications per day
- **Match threshold**: `MATCH_THRESHOLD = 70` - Only apply to jobs >70% skill match
- **Recent jobs only**: Only scrape jobs posted in last 24 hours
- **Duplicate prevention**: Never apply twice to same company+role combo
- **Rate limiting**: Add delays between emails to avoid spam detection

## What NOT to Do

❌ Simple substring matching (use regex word boundaries instead)  
❌ Bare except clauses (catch specific exceptions)  
❌ Hardcoded values (use config.py)  
❌ Mixing print() with logging (use logger module)  
❌ Applying before marking as sent (atomic operations)  
❌ Ignoring email validation errors  

## Testing Expectations

- All new functions should have accompanying tests
- Use pytest for test framework
- Mock external API calls (requests library)
- Test edge cases (empty strings, None values, malformed data)
- Test integration between modules with fixtures

## Performance Considerations

- Batch job processing in waves (10 jobs per wave)
- Add delays between email sends (2-5 seconds)
- Cache company information (don't re-scrape same company)
- Limit API calls to job platforms (respect robots.txt)

## Security Reminders

- Store SMTP password in environment variable only
- Never log sensitive data (passwords, tokens)
- Validate email addresses before sending
- Check for CSV formula injection in output
- Use HTTPS for all external requests
